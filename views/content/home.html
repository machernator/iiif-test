How to test

We divide the test phase into 3 parts

  1. Testing whether the technical access of the application works for you, i.e. whether the application can be called for you and whether all permissions for it are set
  2. Testing the functionality of the application:
    • do all operations within the software work?
    • are the values stored and retrieved correctly or does unexpected behavior or error messages occur?
  3. Testing the stability of the application: do you manage to bypass certain mechanisms of the application, can you somehow crash the application; find out "what you can do". Probably the most amusing part ðŸ˜Š
